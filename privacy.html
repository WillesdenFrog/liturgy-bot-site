What Liturgy Bot is

Liturgy Bot is a private automated service that generates a weekly Children's Liturgy of the Word document for a single Catholic parish in the Diocese of Westminster. It is developed and operated by Romain Thierry. It is not a public-facing application and has no external users.

What Google user data is accessed

Liturgy Bot accesses a single Google account via the Gmail API with one scope:

gmail.send — used to send the weekly liturgy document as an email attachment to two parish catechists, and to send error or conflict notification emails to the developer.

How the data is used

The Gmail API is used exclusively to send outbound emails. Liturgy Bot does not read, modify, delete, or store any email content. It does not access contacts, calendars, Drive, or any other Google service. The only data transmitted is the outbound email itself (a Word document attachment and a short message body).

What data is stored

Liturgy Bot stores an OAuth 2.0 refresh token in Google Cloud Secret Manager to authenticate with the Gmail API. No email content, user data, or personal information is stored, logged, or retained beyond the token required for authentication.

Data sharing

Liturgy Bot does not share any Google user data with third parties. The generated liturgy document is sent only to two designated recipients whose email addresses are stored in Secret Manager.

Data retention and deletion

No Google user data is retained. The OAuth token can be revoked at any time by the account holder through Google Account settings. Revoking the token immediately and permanently disables the bot's access to the Gmail account.

Security

All credentials are stored in Google Cloud Secret Manager with access restricted to a dedicated service account. The application runs in an isolated Cloud Run container. No credentials are stored in code or configuration files.

Contact

For questions or concerns about this privacy policy or Liturgy Bot's use of data, contact liturgybot.admin@gmail.com.
